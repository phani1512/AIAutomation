# 🌐 Web Interface for Selenium SLM

## Overview
A beautiful, interactive web interface for generating Selenium test automation code using the trained Small Language Model (SLM).

## Access the Web Interface

### 1. Start the API Server
```bash
python src/main/python/api_server.py
```

### 2. Open in Browser
Navigate to: **http://localhost:5002**

The web interface will automatically connect to the API and display the connection status.

---

## Features

### 🎨 Modern UI
- **Gradient Design** - Beautiful purple gradient theme
- **Responsive Layout** - Works on desktop, tablet, and mobile
- **Real-time Status** - Connection indicator with live updates
- **Statistics Dashboard** - Track total requests, tokens generated, and response times

### ✨ Three Generation Modes

#### 1. **Generate Code** Tab
Generate complete Selenium test code from natural language prompts.

**Example Prompts:**
- "click login button"
- "enter username in input field"
- "select country from dropdown"
- "verify page title"

**Quick Start Examples:**
Click any of the pre-loaded example buttons to see instant results!

#### 2. **Locator Suggestions** Tab
Get AI-powered locator suggestions from HTML snippets.

**Example Input:**
```html
<button id="submit-btn" class="btn-primary">Submit</button>
```

**Output:**
- Element type detection
- Recommended action
- Prioritized locator strategies
- Full code generation

#### 3. **Action Suggestions** Tab
Get recommended Selenium actions based on element type and context.

**Inputs:**
- **Element Type**: button, input, select, link, checkbox, radio, textarea
- **Context** (optional): login form, navigation menu, search bar, etc.

**Output:**
- Suggested actions (click, sendKeys, select, etc.)
- Generated Selenium code
- Context-aware recommendations

## Usage Examples

### Example 1: Generate Login Test
1. Go to **Generate** tab
2. Enter: `click login button and enter credentials`
3. Click "🚀 Generate Code"
4. Copy the generated code using "📋 Copy to Clipboard"

### Example 2: Find Best Locator
1. Go to **Locator** tab
2. Paste your HTML element:
   ```html
   <input type="email" id="userEmail" name="email" class="form-control">
   ```
3. Click "🎯 Suggest Locators"
4. Review prioritized locator strategies

### Example 3: Get Action Recommendations
1. Go to **Action** tab
2. Select element type: `input`
3. Enter context: `login form`
4. Click "💡 Suggest Actions"
5. See recommended actions: sendKeys, clear, etc.

## API Integration

The web interface connects to these API endpoints:

### Health Check
```javascript
GET http://localhost:5002/health
```

### Generate Code
```javascript
POST http://localhost:5002/generate
{
  "prompt": "your prompt here",
  "max_tokens": 50,
  "temperature": 0.7
}
```

### Suggest Locator
```javascript
POST http://localhost:5002/suggest-locator
{
  "html": "<your HTML element>"
}
```

### Suggest Action
```javascript
POST http://localhost:5002/suggest-action
{
  "element_type": "button",
  "context": "login form"
}
```

## Troubleshooting

### "API Server Not Running" Message
**Solution:**
```bash
# Start the API server
python src/main/python/api_server.py

# Verify it's running
curl http://localhost:5002/health
```

### CORS Errors
The API server has CORS enabled. If you still see CORS errors:
1. Check that Flask-CORS is installed: `pip install flask-cors`
2. Verify CORS is enabled in `api_server.py`

### Slow Response Times
The n-gram model runs on CPU. For faster inference:
1. Use shorter prompts
2. Reduce max_tokens parameter
3. Lower temperature value
4. Consider upgrading to GPU-based model

### Empty or Garbled Output
The model may need more training data:
1. Add more examples to datasets in `src/main/resources/`
2. Re-tokenize: `python src/main/python/tokenize_dataset.py`
3. Re-train: `python src/main/python/train_simple.py`
4. Restart API server

## Performance Tips

### Optimize Model Loading
The model loads once on server startup. Keep the server running to avoid reload time.

### Cache Results
Consider implementing client-side caching for repeated prompts:
```javascript
const cache = new Map();
if (cache.has(prompt)) {
  return cache.get(prompt);
}
```

### Batch Requests
For multiple prompts, consider creating a batch endpoint.
